## Overview

This directory contains AitherZero orchestration playbooks. **Only CI/CD playbooks** are kept in the main directory. Utility playbooks are located in `_utilities/` and deprecated playbooks are in `_deprecated/`.

## [TARGET] CI/CD Playbooks

These playbooks are used by GitHub Actions workflows:

- **`ci-pr-validation.psd1`** - Complete PR validation pipeline (build, test, report)
- **`ci-pr-build.psd1`** - PR build phase (validation and package creation)
- **`ci-pr-test.psd1`** - PR test phase (tests and quality analysis)
- **`ci-pr-report.psd1`** - PR report phase (dashboard, changelog, PR comment)
- **`ci-release.psd1`** - Release pipeline
- **`ci-branch-ecosystem.psd1`** - Complete branch ecosystem (build, test, package, Docker, dashboard, reports)
- **`ci-jekyll-deploy.psd1`** - Jekyll site deployment

## [NOTE] Usage

### Run a CI/CD Playbook Locally

```powershell
# Import module
Import-Module ./AitherZero.psd1 -Force

# Run complete PR validation
Invoke-AitherPlaybook -Name ci-pr-validation

# Run individual phases
Invoke-AitherPlaybook -Name ci-pr-build
Invoke-AitherPlaybook -Name ci-pr-test
Invoke-AitherPlaybook -Name ci-pr-report
```

### Playbook Structure

Playbooks use PowerShell Data (.psd1) format and include:

- **Sequence** - Scripts to execute in order
- **Variables** - Environment variables for scripts
- **Options** - Execution options (parallel, concurrency, etc.)
- **SuccessCriteria** - Criteria for playbook success
- **Artifacts** - Expected output artifacts
- **Reporting** - Report generation configuration

## Other Directories
